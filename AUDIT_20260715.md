# TradingMapClaw third-round audit handoff

Date: 2026-07-15

The supplied Hermes Round 2 report has been cross-reviewed against both public repositories. Mac-only implementation claims are not promoted to independently verified status until a redacted snapshot is generated from the actual Mac directories.

## Run on the Mac

```bash
cd ~/TradingMapClaw/TradingMapClaw
python3 tools/mac_snapshot_audit.py --compile
```

Defaults match the installed layout:

- `~/.hermes/`
- `~/.hermes/scripts/`
- `~/.hermes/cron/scripts/`
- `~/.hermes/cron/reports/`
- `~/.hermes/skills/`
- `~/shared_context/`
- `~/TradingMapClaw/tmc-shared-context`
- `~/TradingMapClaw/tradingmapclaw-website`

Outputs are written to:

- `~/shared_context/mac_audit_snapshot.json`
- `~/shared_context/mac_audit_snapshot.md`

The snapshot redacts API keys, tokens, task bodies and remote URLs. It reports model values, key presence booleans, file hashes, status counts and static code checks.

Static Council and browser-injection checks intentionally remain `WARN`: Council requires a fixture proving R2 consumes R1 and R3 only runs on disagreement; Zhipu injection requires a logged real-browser run plus cleanup evidence. Source-pattern matching alone is not accepted as proof of runtime behavior.

## Findings already established from public source

- The website quality gate and unit tests were operational before the third-round changes.
- The candidate checker was a deterministic schema/compliance preflight, not an independent external market-data fetch.
- LIVE freshness and independent-source-domain enforcement needed hardening.
- The Hermes report contradicts itself on current cron counts.
- Public documentation previously reported 230+ scripts while the current Mac inventory was described as 415+; current public wording is now `hundreds`, and exact canon waits for the generated snapshot.
- The connected Codex identity has write/admin access to both public repositories, but could not see the private shared-context repository in this review environment.
- The hardened website baseline passes 14 unit tests, the workspace contract and idempotent build, plus `verify-all` at 26 PASS / 0 FAIL / 0 WARN / 3 network SKIP.

## Required Mac remediation after snapshot

1. Make every Council round pass its model/provider explicitly; do not set a DeepSeek default that can silently affect unrelated calls.
2. Unit-test R2 with R1 output in the prompt and R3 behind a real disagreement conditional.
3. Add bounded timeout/retry/fallback behavior for R1 and reject malformed vote payloads.
4. Wrap browser temp-file injection in `try/finally`; remove the file with `os.unlink`; keep the job disabled until a successful logged run.
5. Resolve ambiguous duplicate default-model variables or document exact consumers and precedence.
6. Generate the authoritative cron inventory and update public facts only from that snapshot.

## Public publishing design

The website repository now contains a zero-additional-token publication lane for deterministic system updates and a weekly Verification Ledger lane for research-process evidence. System updates may auto-merge after CI; any research ledger stops at a reviewable PR.
