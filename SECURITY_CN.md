# 安全说明

## 安全边界

TradingMapClaw（TMC）在严格且刻意受限的安全边界内运行。系统采用"最小能力"哲学设计——只有它需要的权限，没有多余。

### 无券商 API — WATCHLIST_ONLY

TMC 是研究工具。不执行交易。

- 系统中任何位置都不存储券商 API 凭证。
- 232 个 Python 脚本中没有任何交易执行能力。
- 系统不能下单、改单或撤单。
- 这是架构约束，不是配置选项——没有"交易模式"可以开启。

### 无用户数据采集

- 系统从 12+ 公开来源采集市场数据。不采集、存储或传输运营者以外的任何用户数据。
- 无遥测、无分析、无追踪。
- 无云账户或第三方 SaaS 数据存储依赖。

### 纯本地存储

- 所有数据文件、配置和凭证存储在单台 Mac mini 本地。
- 不使用云存储（AWS S3、Google Cloud、Azure）。
- 无数据库暴露在网络中。

### 代理边界

- 所有出站网络流量经单一代理 `127.0.0.1:10808`。
- 采集脚本不建立直接互联网连接。
- 代理提供单一网络检查和调试咽喉点。

## 安全实践

### Codex 代码审计

- 230 个文件经 Codex 代码审查扫描。
- 0 个高危 bug。
- 425/425 脚本通过 `py_compile` 语法检查。
- 已知技术债（17 个裸 except、61 个硬编码路径、36 个未关闭文件句柄、36 个 subprocess 无 timeout）记录在封版日志中。这些均不代表可利用的漏洞——是代码质量问题，留待逐步清理。

### 工程宪法

所有 AI 辅助代码修改受 10 条工程宪法约束（派生自 Karpathy 的 CLAUDE.md）：

1. 改前先读——理解现有代码再修改。
2. 先想再写——动手前说明意图和假设。
3. 简单——只写解决眼前问题的最小代码。
4. 外科手术式修改——最小 diff，不顺带重构。
5. 验证——修 bug 前先写会失败的测试。
6. 目标驱动——动手前定义成功标准。
7. 调试——先调查别猜，读完整报错，一次改一处。
8. 依赖——无理由不加新依赖。
9. 沟通——说明改了什么、为什么、哪里不确定。
10. 警惕失败模式——发现 Kitchen Sink / Wrong Abstraction / Happy Path / Runaway Refactor 立刻停。

此宪法为文件级强制（`skills/engineering_constitution.md`），优先级高于单次任务便利性。

### 封版模块

四个关键模块已封版——未经运营者明确批准不可修改：

- `send_reports`——推送交付管道
- `runtime_guard`——系统健康监控
- `scheduler`——cron 任务执行引擎
- `stocks.yaml` schema——标的覆盖配置

### 凭证隔离

- API 密钥和 token 存储在环境变量和本地配置文件中。
- 凭证不提交至版本控制。
- GitHub 仓库不含 API 密钥、token、账号号码或个人财务数据。
- 脚本使用硬编码本地路径（61 处，作为技术债追踪），但这些路径不含密钥。

### 预算看门狗

- 月度 API 支出上限 ¥400 人民币。
- 预算看门狗实时监控支出，在接近上限时暂停非必要 API 调用。
- 防止 bug 或配置错误导致的失控成本。

## 漏洞报告方式

如果你认为你发现了 TradingMapClaw 的安全漏洞：

1. **不要公开开 GitHub Issue。** 安全漏洞在修复前不应公开披露。
2. **通过 [GitHub Security Advisory](https://github.com/mickeyweijiajun-web/TradingMapClaw/security/advisories/new) 私下报告。**
3. 请包含：
   - 漏洞描述
   - 复现步骤（如适用）
   - 潜在影响
   - 修复建议（可选）

### 响应时间

维护者（Mickey Wei）有身体残疾（右臂臂丛神经撕脱、终身回肠造口），行动和打字速度受限。他会尽快确认收到，但响应时间可能慢于典型开源项目。这是单人维护的系统——没有安全团队。

### 范围

在范围内：
- Python 脚本中可能导致数据泄露、凭证泄漏或系统被入侵的漏洞。
- 推送管道（Telegram/飞书）中可能导致报告发错人的问题。
- 质量门中可能导致畸形数据未被检测而进入报告的 bug。

不在范围内：
- 系统不能执行交易（这是设计，不是漏洞）。
- yfinance 限流或 Yahoo Finance 数据准确性（第三方问题）。
- LLM 输出质量或准确性（研究限制，非安全问题）。
- 系统缺乏实时 Level 2 数据（架构约束）。
