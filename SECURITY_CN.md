# 安全说明

> **TradingMapClaw（TMC）v2.0 | 2026-07-12**

## 安全边界

TradingMapClaw（TMC）在严格且刻意受限的安全边界内运行。系统采用"最小能力"哲学设计——只有它需要的权限，没有多余。

### 无券商 API — WATCHLIST_ONLY

TMC 是研究工具。不执行交易。

- 系统中任何位置都不存储券商 API 凭证。
- **230+ 个** Python 脚本中没有任何交易执行能力。
- 系统不能下单、改单或撤单。
- 这是架构约束，不是配置选项——没有"交易模式"可以开启。

### 无用户数据采集

- 系统从 12+ 公开来源采集市场数据。不采集、存储或传输运营者以外的任何用户数据。
- 无遥测、无分析、无追踪。
- 无云账户或第三方 SaaS 数据存储依赖。

### 纯本地存储

- 所有数据文件、配置和凭证存储在单台 Mac mini 本地。
- 不使用云存储（AWS S3、Google Cloud、Azure）。
- 无数据库暴露在网络中。

### 代理边界

- 所有出站网络流量经单一代理 `127.0.0.1:10808`。
- 采集脚本不建立直接互联网连接。
- 代理提供单一网络检查和调试咽喉点。

## 安全实践

### 双引擎交叉审计

- 全部脚本经 Hermes+Codex 交叉审计扫描。
- **230+/230+ 脚本通过 `py_compile` 语法检查。**
- 两轮 Hermes+Codex 联合审计共发现并修复 **14 个 bug**——其中包括导致 Pass C 记忆功能失效的严重 `HERMES_HOME` 未定义变量 bug。完整清单见 [CHANGELOG_CN.md](CHANGELOG_CN.md)。
- 在 v2.0 封版前，Hermes+Codex 联合的**周一就绪审计**覆盖 7 个运营风险点，全部 7/7 通过（PASS 或 FIXED）。

### 凭证隔离

- API 密钥和 token 存储在环境变量和本地配置文件中。
- 凭证不提交至版本控制。
- GitHub 仓库不含 API 密钥、token、账号号码或个人财务数据。

### 冻结模块

四个关键模块已封版——未经运营者明确批准不可修改：

- `send_reports.py`——推送交付基础设施（1360 行）
- `runtime_guard`——系统健康监控
- `scheduler`——cron 任务执行引擎
- `stocks.yaml` schema——标的覆盖配置

### 预算看门狗

月度 API 支出上限 **55 美元（¥400）**持续强制执行，检查点为北京时间 23:00，在接近上限时暂停非必要 API 调用，防止 bug 或配置错误导致的失控成本。实际支出约为每月 7 美元（占上限的 13.5%）。跟踪器从 `codex_usage.jsonl` 和 `quality_scores.csv` 聚合真实成本——v1.6.1 的修复细节见 [CHANGELOG_CN.md](CHANGELOG_CN.md)。

### 4 级数据回退链

每个数据字段都要经过直接 API → 代理路由 → Python 库 → 本地缓存，才会被标记为缺失。这降低了在数据压力下绕过代理边界或访问未记录端点的动机。完整细节见 [ARCHITECTURE_CN.md](ARCHITECTURE_CN.md#7-数据源与回退链)。

## 漏洞报告方式

如果你认为你发现了 TradingMapClaw 的安全漏洞：

1. **不要公开开 GitHub Issue。** 安全漏洞在修复前不应公开披露。
2. **通过 [GitHub Security Advisory](https://github.com/mickeyweijiajun-web/TradingMapClaw/security/advisories/new) 私下报告。**
3. 请包含：
   - 漏洞描述
   - 复现步骤（如适用）
   - 潜在影响
   - 修复建议（可选）

### 响应时间

维护者（Mickey Wei）有身体残疾（右臂臂丛神经撕脱、终身造口），行动和打字速度受限。他会尽快确认收到，但响应时间可能慢于典型开源项目。这是单人维护的系统——没有安全团队。

### 范围

在范围内：
- Python 脚本中可能导致数据泄露、凭证泄漏或系统被入侵的漏洞。
- 推送管道（Telegram/飞书）中可能导致报告发错人的问题。
- 质量门中可能导致畸形数据未被检测而进入报告的 bug。

不在范围内：
- 系统不能执行交易（这是设计，不是漏洞）。
- yfinance 限流或 Yahoo Finance 数据准确性（第三方问题）。
- LLM 输出质量或准确性（研究限制，非安全问题）。
- 系统缺乏实时 Level 2 数据（架构约束）。
